# After-task — v0.1.0 release consolidation (2026-05-31)

**Owner:** Shannon (Ada overview · Rose claim-vs-evidence · Curie recovery ·
Karpinski no-regression).

## What shipped (cumulative, on `main`)

The `drm()` / `bf()` distributional-regression front end, drmTMB-mirroring
syntax, all recovery-tested and merged one-PR-per-slice with green CI:

- **Gaussian surface (complete):** univariate & bivariate location–scale +
  residual `rho12`; random effects on the mean (intercept `(1|g)`, slope
  `(0+x|g)`, correlated `(1+x|g)`, crossed/nested `(1|g)+(1|h)`) **and the scale**
  (`sigma ~ (1|g)` via Gauss–Hermite); structured effects `relmat` / `animal` /
  `phylo` / `spatial`; `meta_V`; Wald + **profile** + parametric **bootstrap**
  intervals; `predict` / `simulate` / `fitted` / `residuals`.
- **8 families:** Gaussian, Student-t, Poisson, NegBinomial2,
  TruncatedNegBinomial2, Beta, Gamma, LogNormal.
- **Count modifiers:** `zi` (ZIP / ZINB), `hu` (hurdle).
- **Webpage:** DocumenterVitepress (the docs.makie.org look, Node via
  `NodeJS_jll` — no system install / no extra CI step) + CairoMakie figures
  incl. the **Confidence Eye**; executed examples; honest per-page status tags.

## Rose audit (claim-vs-evidence)

- **No overclaim of family parity.** Families are validated by **simulation
  parameter recovery** and mirror drmTMB *syntax*; the numerical drmTMB-parity
  gate (RCall vs. drmTMB v0.1.3 outputs) is explicitly **pending (#17)** in NEWS,
  README, and the model-map.
- **Engine claims** (q=4 PLSM: 2.18× over drmTMB, O(p) to p=10,000, valid CIs)
  are HANDOVER-verified and unchanged — kept as the "foundation", not attributed
  to the new families.
- **License:** MIT; no drmTMB GPL source vendored. Parity uses generated outputs
  only (and the gate itself is still pending).
- **Honesty on scope:** `src/experimental/` (REML, infer_q4, EM), the bivariate-
  phylo q=4 public front end, and the R↔Julia bridge are flagged **not yet
  wired** in README / ROADMAP.
