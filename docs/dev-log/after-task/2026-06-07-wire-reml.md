## Scope

Expose **opt-in REML estimation** through the public fit verb,
`drm(...; method = :REML)`, with **ML staying the default**. Landed **Slice 2**
(univariate / location-scale Gaussian REML, independent of the front end #187);
**deferred Slice 1** (the q=4 bivariate REML in `experimental/reml_q4.jl`, which
rides on the public q=4 path gated by #187). The shared API contract — the
`method` kwarg, the `DrmFit` REML metadata, and the model-selection guard — is
built once here and is what Slice 1 will reuse when #187 lands.

## The model + estimator

Fixed-effect Gaussian location–scale `y_i ~ N(Xμ_i β_μ, σ_i²)`,
`log σ_i = Xσ_i β_σ`. REML profiles β_μ out by weighted least squares
(`W = diag(σ_i⁻²)`) and optimises the restricted objective

    ℓ_R(β_σ) = ℓ_P(β̂_μ(β_σ), β_σ) − ½ logdet(Xμ'WXμ) + (pμ/2) log 2π

over β_σ; β̂_μ is the final WLS estimate. The `−½ logdet` term corrects the scale
for the pμ degrees of freedom spent on β_μ, so the REML residual variance is
LARGER than ML — in the homoscedastic case exactly `σ²_REML = SSR/(n−pμ)` vs
`σ²_ML = SSR/n` (ratio `n/(n−pμ)`).

## Changes

- `src/gaussian_core.jl`
  - `DrmFit` gains `estim_method::Symbol`, `reml_loglik::Float64`,
    `ml_loglik::Float64`. The 11-arg outer constructor defaults them to
    `:ML / NaN / loglik`, so **every existing fitter is unaffected**. New
    `_withreml(fit, reml_ll, ml_ll)` helper; `_withformula`/`_withnll`/`_withranef`
    forward the new fields.
  - `drm(::DrmFormula, ::Gaussian; …, method::Symbol = :ML)` — validates `method`,
    routes `:REML` on the fixed-effect cell to `_fit_fixed_gaussian_reml`, and
    rejects unsupported cells (RE / structured / phylo / meta / bivariate) under
    `:REML` with a clear `ArgumentError`.
  - `_fit_fixed_gaussian_reml` — the REML fitter (WLS-profiled β_μ; LBFGS over
    β_σ with ForwardDiff; block-diagonal vcov: `(Xμ'WXμ)⁻¹` for β_μ, FD-Hessian
    inverse for β_σ; attaches an ML-layout `nll_full` for profile intervals).
  - Accessors `estimation_method`, `reml_loglik`, `ml_loglik`; `loglik` docstring
    now carries the REML cross-structure caveat.
  - `aic`/`bic` emit a one-time `@warn` on a REML fit (variance-only-comparison
    caveat) via `_reml_infocrit_warn`.
- `src/comparison.jl` — `_mean_structure` + `_reml_compare_guard`; `lrtest`
  (and thus `anova`) **error** on a cross-mean-structure REML comparison, silent
  otherwise.
- `src/DRM.jl` — export `reml_loglik`, `ml_loglik`, `estimation_method`.
- `test/test_reml.jl` (wired into `runtests.jl`) — ML-unchanged, REML ≥ ML
  (closed-form ratio), FD-gradient self-consistency, the guard (both directions),
  and unsupported-cell rejections.

## Acceptance (targets, pending CI)

- **ML unchanged:** `method = :ML` ≡ no kwarg, coef + loglik byte-for-byte.
- **REML ≥ ML:** `σ²_REML > σ²_ML`, ratio `≈ n/(n−pμ)`; β̂_μ unchanged.
- **FD gradient:** `max|∇ℓ_R| < 1e-4` at the optimum.
- **Guard:** cross-mean REML `lrtest`/`anova` throws; variance-only + all-ML pass.
